# Frozen S2-4b Diagnosis Baseline Repair Plan

> **For Hermes:** Execute this plan as the main/default orchestrator. Use read-only specialists for evidence/audit when useful; keep code writes serial and verify all specialist claims locally.

**Goal:** Repair generic row-recovery capabilities discovered from the frozen S2-4b diagnosis baseline residuals, then rerun a new system-registered diagnosis baseline from the same frozen S2-4b raw results and compare the new Stage5 final table against locked GT.

**Architecture:** The S2-4a/S2-4b semantic boundary is frozen. The repair loop must reuse the already captured S2-4b raw responses and replay only deterministic downstream steps: S2-5 parsing, S2-6 validation, S2-7 compatibility projection, Stage3 relation materialization, Stage5 final table closure, and final-table-vs-GT diagnosis compare. Residual papers such as `5GIF3D8W` and `QLYKLPKT` are validation cases for generic capabilities only; runtime behavior must not hard-code paper keys or GT-target counts.

**Tech Stack:** Python stdlib, project `unittest`, maintained PLGA Stage2/Stage3/Stage5 scripts, TSV/JSONL artifacts under `data/results/`.

---

## Non-negotiable constraints

1. **Diagnosis-only language.** Do not call current outputs benchmark-valid/invalid or benchmark-blocked. In this phase, `baseline` means a system-registered diagnosis baseline.
2. **Frozen S2-4b reuse.** Do not make fresh live LLM calls. Reuse the prior frozen S2-4b raw-response set unless the user explicitly authorizes a new live run.
3. **Generic repairs only.** Do not add `paper_key == ...` runtime special cases, GT-count-targeting, or row-count patches. Specific papers are tests/evidence cases only.
4. **Source/GT/current artifact triage before changing behavior.** For every residual, inspect source text/table evidence, locked GT authority, old baseline artifacts, and new replay artifacts before deciding the repair class.
5. **No recency-based source selection.** Resolve all workflow inputs through explicit paths or `data/results/ACTIVE_RUN.json`; never choose by newest directory, sort order, glob-first, or modification time.
6. **Mainline code before baseline rerun.** The new diagnosis baseline must be generated by the repaired mainline code, not by ad hoc artifact editing.
7. **Clean patch discipline.** Do not merge unrelated dirty working-tree changes with this repair. Isolate source/tests/docs needed for this loop.
